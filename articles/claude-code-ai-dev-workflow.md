## はじめに

「AIを開発に使う」と言っても、ChatGPTにコードを貼り付けてコピペする段階はもう過去の話です。2024年後半から急速に普及した**AIコーディングエージェント**は、開発者のワークフローそのものを変えつつあります。

私たちPRIでも、Anthropicが提供する **Claude Code** を実際のプロジェクトに導入しました。本記事では、導入の背景から具体的な活用方法、そして現場で感じたメリットと注意点までを率直に共有します。

## Claude Code とは

Claude Code は、Anthropicが提供するCLIベースのAIコーディングエージェントです。主な特徴は以下のとおりです。

- **ターミナルネイティブ** — エディタを問わず、普段の開発環境にそのまま組み込める
- **コードベース全体の理解** — プロジェクト構造を自律的に探索し、文脈を把握した上で提案・実装を行う
- **ツール実行** — ファイルの読み書き、シェルコマンドの実行、Git操作まで一貫して対応
- **CLAUDE.md によるカスタマイズ** — プロジェクトごとにルールや慣習を定義し、AIの振る舞いを制御できる

```bash
# インストール
npm install -g @anthropic-ai/claude-code

# プロジェクトディレクトリで起動
cd my-project
claude
```

起動すると対話型のセッションが始まり、自然言語で指示を出すだけでコードの生成・修正・リファクタリングなどを実行してくれます。

## 私たちの活用シーン

### 1. 新機能のプロトタイピング

最も効果を実感しているのが、新機能の初期実装です。

```
「ユーザーのアクティビティログをCSVでエクスポートする機能を追加して。
既存の /api/users/:id/activities エンドポイントのデータを使って。」
```

このような指示だけで、Claude Code はコードベースを探索し、既存のAPIレスポンス構造を理解した上で、エクスポート機能を実装します。ゼロから書く場合と比べ、**プロトタイプの完成までの時間が体感で 1/3 程度**になりました。

### 2. テストコードの生成

テストを書く作業は重要ですが、時間がかかりがちです。Claude Code に既存のコードを読ませた上で「このモジュールのユニットテストを書いて」と依頼すると、エッジケースを含むテストを生成してくれます。

### 3. コードレビューの補助

PRの差分を読ませて「この変更にセキュリティ上の懸念はあるか」「パフォーマンスのボトルネックになりそうな箇所はあるか」といった観点でのレビューを依頼しています。人間のレビューを置き換えるものではありませんが、レビュー前の事前チェックとして有効です。

### 4. ドキュメント・技術記事の執筆

実はこの記事自体も、Claude Code を活用して執筆しています。CLAUDE.md にフォーマットやトーンのルールを定義しておくことで、プロジェクトの文脈を踏まえた記事が生成されます。

## CLAUDE.md の活用

Claude Code を現場で使う上で鍵になるのが `CLAUDE.md` です。これはプロジェクトのルート（またはサブディレクトリ）に配置する設定ファイルで、Claude に対して以下のような指示を与えられます。

```markdown
# CLAUDE.md の例

## コーディング規約
- TypeScript の strict mode を使用する
- エラーハンドリングは Result 型で行う（throw しない）
- テストは Vitest を使用する

## アーキテクチャ
- Clean Architecture に基づくレイヤー構成
- src/domain/ にビジネスロジック、src/infra/ に外部連携
```

プロジェクトごとの慣習をファイルに書いておくだけで、AIが「このプロジェクトではどう書くべきか」を理解してくれます。新しいメンバーのオンボーディングにも役立つ副次効果がありました。

## 導入して見えた注意点

### 生成結果の検証は必須

AIが書いたコードを無条件に信頼するのは危険です。特に以下の点は人間が必ず確認しています。

- **セキュリティ** — 入力バリデーションの漏れ、認証・認可の抜け
- **ビジネスロジック** — 仕様を正しく反映しているか
- **既存コードとの整合性** — 命名規則やアーキテクチャパターンの逸脱

### 機密情報の取り扱い

コードベースの内容がAPIを通じて送信されるため、機密性の高い情報（環境変数、顧客データなど）の取り扱いには注意が必要です。`.claudeignore` を活用して除外対象を明示的に管理しています。

### 「AIに任せすぎない」バランス

便利さゆえに、考える前にAIに投げてしまう癖がつくリスクがあります。設計判断やアーキテクチャの意思決定は人間が行い、実装の加速にAIを使う、というバランスが重要です。

## まとめ

Claude Code の導入により、私たちの開発ワークフローは確実に変化しました。

- **プロトタイピングの高速化** — アイデアから動くコードまでの時間が大幅に短縮
- **定型作業の削減** — テスト生成、ドキュメント作成などの負荷が軽減
- **CLAUDE.md による標準化** — プロジェクトの慣習をコードとして管理

一方で、AIの出力を鵜呑みにしない姿勢、機密情報の管理、人間の判断力の維持といった点は引き続き意識すべきです。

AI開発ツールは「魔法の杖」ではなく「優秀なペアプログラマー」です。うまく付き合うことで、チーム全体の開発体験を向上させることができると実感しています。
